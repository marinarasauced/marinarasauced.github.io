# ROS2 Graph

## Overview
The ROS2 "graph" consists of core ROS2 concepts and describes the terminology and structure of ROS2 at the user-level. These concepts include, but are not limited to:
- Nodes
- Topics
- Messages
- Services
- Parameters
- Actions

## Nodes
Nodes are instances of executables that act as individual computational entities. Each node performs a specific task; e.g., a node might be an instance of a filter that subscribes to sensor measurements and publishes state estimates whereas another node might employ a closed-loop controller that subscribes to the state estimates and publishes motor commands.

Nodes typically consist of combinations of publishers/subscribers, service servers/clients, and action servers/clients, as well as any other code necessary to perform their task.

## Topics

### What are Topics
Topics are named buses that nodes use to exchange data in the form of ROS2 messages. Topics are named as a string in the creation of publishers/subscribers and only exist when they are named in a running node.

![A node subscribes to messages on a topic published by a different node](https://docs.ros.org/en/jazzy/_images/Topic-SinglePublisherandSingleSubscriber.gif)

Each topic is limited to a unique data structure called a message. There can be any number of topics that use the same message types, but for any names topic, only data of the specified message type can be published to the topic. 

Messages are defined as `.msg` files in ROS2 packages and consist of combinations of field types and field names. Field names specify the name of a value in a message and field types specify the type that value. Field types can be either a primitive data type, another ROS2 message type, or an array of a primitive data type or ROS2 message type.

### Messages Structure
ROS2 provides an extensive suite of default message types in addition to any user defined message types. Below is the definition of `TransformStamped.msg`, the message type used for VICON localization:

```msg
# TransformStamped.msg

# message definition
# this expresses a transform from coordinate frame header.frame_id to the coordinate frame child_frame_id.

std_msgs/Header header
string child_frame_id
geometry_msgs/Transform transform
```

This message type consists of a `Header` message that contains the time stamp of the message and frame id, a `string` of the child frame id, and a `Transform` message that contains the translation and rotation.

ROS2 nodes can contain publishers that publish messages to topics and subscribes that subscribe to topics and receive the respective messages in callback functions. Topics are only used in publisher/subscriber communication.
